# 01 GCC Compilation

## Зачем это

Этот урок нужен, чтобы перестать воспринимать `gcc` как магию.

После него ты должен руками различать:
- исходник `.c`
- объектный файл `.o`
- исполняемый файл

И понимать смысл ключей:
- `-E`
- `-S`
- `-c`
- `-o`
- `-Wall -Wextra -g`

## Артефакт урока

Маленькая программа на C, которая:
- компилируется без ошибок;
- сначала проходит стадию `.c -> .o`;
- потом линкуется `.o -> executable`;
- при запуске печатает:

```text
2 + 3 = 5
```

## Что сделать

1. Открой `template.c`.
2. Реализуй функцию `add(int a, int b)`.
3. Сделай `main()`, который вызывает `add(2, 3)` и печатает точную строку:

```text
2 + 3 = 5
```

## Ручные команды

Стадия препроцессора:

```bash
gcc -E template.c | sed -n '1,20p'
```

Стадия ассемблера:

```bash
gcc -S template.c -o template.s
```

Стадия компиляции в объектный файл:

```bash
gcc -Wall -Wextra -g -c template.c -o lesson01.o
```

Линковка объектного файла:

```bash
gcc -Wall -Wextra -g lesson01.o -o lesson01
```

Запуск:

```bash
./lesson01
```

## Автопроверка

По умолчанию `test.sh` проверяет эталонное решение.

Чтобы проверить свою версию:

```bash
./test.sh template.c
```

Если ты работаешь в отдельном файле, передай его первым аргументом:

```bash
./test.sh my_work.c
```

Если `gcc` недоступен, можно явно указать компилятор:

```bash
CC=clang ./test.sh my_work.c
```

## Критерий прохождения

Урок можно считать пройденным, если ты можешь:
- скомпилировать файл в `.o` через `-c`;
- собрать executable из `.o`;
- объяснить разницу между `.c`, `.o` и бинарником;
- объяснить, что делает `-Wall -Wextra -g`;
- получить точный вывод `2 + 3 = 5`.

## Минимальный смысл

- `-E` = только препроцессор
- `-S` = получить ассемблерный текст
- `-c` = получить объектный файл, без линковки
- без `-c` на финальном шаге ты получаешь исполняемый файл

Если это ещё не ощущается как система, не расширяй урок. Просто дожми один проход до конца.
