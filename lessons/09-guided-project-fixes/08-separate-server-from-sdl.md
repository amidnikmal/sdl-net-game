# 08 Separate Server From SDL

## Проблема

Серверный код тянет за собой клиентские SDL-зависимости через общие структуры.

Это плохая граница между слоями.

## Где смотреть

- `src/cl_player.h`
- `src/serv_state.h`
- связанные include-цепочки

## Сначала пройти

- `lessons/08-c-architecture-atoms/02-opaque-handle`
- `lessons/08-c-architecture-atoms/05-domain-vs-transport`
- `lessons/08-c-architecture-atoms/06-no-globals-context`

## Что нужно сделать

Выделить чистую серверную/сетевую модель игрока без SDL-полей.

Цель:
- сервер не должен знать про `SDL_Surface`, `TTF_Font` и клиентскую графику.

## Чего НЕ делать

- Не переделывать весь проект в новую идеальную архитектуру.
- Не ломать сразу все клиентские вызовы.
- Не смешивать это с полным рефакторингом ownership.

## Минимальный критерий готовности

Есть отдельная структура данных, достаточная серверу для работы с сетью и состоянием, без SDL-зависимостей.
