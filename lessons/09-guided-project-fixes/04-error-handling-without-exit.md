# 04 Error Handling Without Exit

## Проблема

Сетевые helper-ветки слишком рано решают судьбу всего процесса через `exit()`.

Это плохая граница ответственности.

## Где смотреть

- `src/network.c`
- `src/server.c`
- при необходимости `src/cl_network.c`

## Сначала пройти

- `lessons/03-sockets-udp`
- `lessons/08-c-architecture-atoms/03-error-return`

## Что нужно сделать

Поменять контракты так, чтобы helper:
- возвращал статус;
- а решение “падать/не падать” принимал вызывающий слой.

## Чего НЕ делать

- Не размазывать обработку ошибок по всему проекту сразу.
- Не чинить параллельно все другие проблемы памяти.
- Не делать большой системный рефакторинг логирования.

## Минимальный критерий готовности

Хотя бы один ключевой сетевой helper больше не вызывает `exit()` напрямую, а возвращает код ошибки.

Ты можешь объяснить:
- почему `send_packet()` не должен решать за весь процесс;
- где должен находиться policy-слой завершения программы.
