# 05 Address Semantics

## Проблема

В клиентском коде спутаны:
- адрес, на котором слушает сервер;
- адрес назначения клиента;
- смысл `INADDR_ANY`.

Из-за этого код работает только в частном случае и вводит в заблуждение.

## Где смотреть

- `src/cl_network.c`
- `src/server.c`

## Сначала пройти

- `lessons/03-sockets-udp`
- активные Anki микроатомы про `INADDR_ANY`

## Что нужно сделать

Сделать семантику адресов явной:
- сервер bind'ится на `INADDR_ANY`;
- клиент шлёт в конкретный адрес назначения.

## Чего НЕ делать

- Не превращать задачу в конфигурационную систему.
- Не пытаться сразу поддержать все режимы сети.
- Не трогать пока disconnect, heartbeat и другие протокольные края.

## Минимальный критерий готовности

В клиентском коде больше нет смысла “адрес назначения = `INADDR_ANY`”.

Ты можешь словами объяснить:
- что означает `INADDR_ANY` у сервера;
- почему это не адрес peer'а для клиента.
