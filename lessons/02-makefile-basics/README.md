# 02 Makefile Basics

## Зачем это

После `01-gcc-compilation` следующий шаг — перестать руками повторять длинные команды сборки.

Задача этого урока:
- вынести команды в `Makefile`;
- увидеть разницу между целью, зависимостью и рецептом;
- руками использовать:
  - переменные
  - `$@`
  - `$<`
  - `$^`
  - `.PHONY`

## Артефакт урока

Маленький проект из двух `.c` файлов, который собирается через `make` и печатает:

```text
2 + 3 = 5
```

## Что сделать

1. Возьми шаблон из директории `template/`.
2. Дополни `template/Makefile` в местах `TODO`.
3. Собери проект через `make`.
4. Запусти бинарник.
5. Проверь, что `make clean` удаляет артефакты сборки.

## Что здесь важно понять

- `CC` — какой компилятор используется
- `CFLAGS` — флаги компиляции C
- `$@` — имя текущей цели
- `$<` — первая зависимость
- `$^` — все зависимости
- `.PHONY` — цель, которая не является настоящим файлом

## Минимальная практика

Сборка шаблона/своей версии:

```bash
make -C template
```

Запуск:

```bash
./template/app
```

Очистка:

```bash
make -C template clean
```

## Автопроверка

По умолчанию `test.sh` проверяет эталон в `solution/`.

```bash
./test.sh
```

Чтобы проверить свою директорию проекта:

```bash
./test.sh template
```

Если нужен другой компилятор:

```bash
CC=clang ./test.sh template
```

## Критерий прохождения

Урок можно считать пройденным, если ты можешь:
- объяснить, что делает `make all`;
- объяснить разницу между `$@`, `$<`, `$^`;
- собрать проект командой `make`;
- удалить артефакты командой `make clean`;
- получить точный вывод `2 + 3 = 5`.

## Связь с основным проектом

После урока сравни этот `Makefile` с корневым `Makefile` репозитория и ответь себе на 2 вопроса:
- почему имя `CPP_FLAGS` там слабое и чем лучше `CFLAGS` / `CPPFLAGS`;
- почему цель вроде `build` и `clean` лучше явно помечать как `.PHONY`.
