# 05 SDL Graphics

## Зачем это

Этот урок нужен, чтобы пройти минимальную графическую механику SDL 1.2 без лишней сложности окна и игрового цикла.

Цель:
- создать `SDL_Surface`;
- закрасить поверхность через `SDL_FillRect`;
- использовать `SDL_Rect` как позицию назначения;
- перенести одну поверхность на другую через `SDL_BlitSurface`;
- проверить результат не “на глаз”, а через чтение пикселей.

## Почему без окна

В основном проекте есть реальный экран и `SDL_SetVideoMode`, но для первого графического урока это лишний шум.

Здесь мы делаем off-screen стенд:
- `canvas` = белый фон;
- `sprite` = маленький красный прямоугольник;
- потом blit'им `sprite` на `canvas`.

Так можно изолированно понять surfaces и rects.

## Артефакт урока

Программа должна вывести:

```text
inside: 255 0 0
outside: 255 255 255
```

Смысл:
- `inside` = пиксель внутри области, куда был blit;
- `outside` = пиксель на фоне, который не затронут.

## Что сделать

1. Открой `template.c`.
2. Заполни `TODO`, чтобы:
   - инициализировать SDL;
   - создать `canvas` и `sprite`;
   - закрасить `canvas` белым;
   - закрасить `sprite` красным;
   - blit'нуть `sprite` на `canvas` в точку `(2, 1)`;
   - прочитать два пикселя и вывести их RGB.

## Что здесь важно понять

- `SDL_Surface` — это поверхность в памяти;
- `SDL_Rect` задаёт позицию и/или область;
- `SDL_FillRect` заливает цветом;
- `SDL_BlitSurface` копирует пиксели из одной поверхности в другую;
- проверять графику можно не только глазами, но и чтением данных.

## Ручная компиляция

Если в системе есть `sdl-config`:

```bash
gcc -Wall -Wextra -g template.c -o lesson05 $(sdl-config --cflags --libs)
SDL_VIDEODRIVER=dummy ./lesson05
```

Если вместо `sdl-config` используется `pkg-config`:

```bash
gcc -Wall -Wextra -g template.c -o lesson05 $(pkg-config --cflags --libs sdl)
SDL_VIDEODRIVER=dummy ./lesson05
```

## Критерий прохождения

Урок можно считать пройденным, если ты можешь объяснить:
- чем `SDL_Surface` отличается от “окна”;
- зачем нужен `SDL_Rect dst_rect` в `SDL_BlitSurface`;
- почему пиксель `(2, 1)` после blit красный;
- почему пиксель `(0, 0)` остаётся белым;
- зачем в headless-среде может пригодиться `SDL_VIDEODRIVER=dummy`.
